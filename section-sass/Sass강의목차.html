SASS Basics

Pre-processing 이란?

'pre'란 '앞'(前)이고, 'processing'은 처리(處理)한다는 뜻이다. 따라서 preprocessor이란 단어의 뜻은, 목적하는 어떤 결과물을 '미리 Processing해 주는' 처리기라 할 수 있다. 같은 이치로 SASS(Syntactically 'Awesome' Style Sheets) Compiler는 사용자가 작성하는 SASS문서(확장자 .scss)를 compiler가 미리 처리하여 결과물인 CSS(확장자 .css) 파일을 생성해 주는 역할을 한다. 하지만 단순히 CSS 파일을 생성하는데 그치지 않고, CSS에서는 제공하지 않는 기능을 사용하여 직접 CSS 파일을 생성하는 것과 비교할 수 없을 만큼의 부가적으로 제공하여 단순 반복이라는 지루한 작업에서 우리를 해방시켜 주는, 그야 말로 'awesome'한 Utility라 할 수 있다.

이미 CSS를 쓰고 있는 사람이나, CSS 초보자도 SASS를 당장 쓸 수 있나?

결론 부터 말하면, 당연히 쓸 수 있다. 왜냐하면, SASS도 CSS의 문법을 똑 같이 사용하기 때문이다. 따라서 기존 사용자가 지금 쓰는 CSS를 모두 없애고 다시 해야 된다든지, 초보자의 경우 SASS 문법을 배우고 난 후에나 사용할 수 있다든지 하는 걱정은 할 필요없다.

이제 그 첫 번째 단계인 VScode를 설치와 Live SASS compiler 확장을 설치하도록 하자.

VSCode 설치하기

Visual Studio Code 홈페이지로 가 보자.

VSCode의 종류

VSCode는 두 종류의 설치 프로그램이 있다.

  • Stable version : 안정화 버전
    • System install : OS 사용자 모두 사용
    • User install : 특정 사용자만 사용
  • Insiders version : 테스트 버전
    • User install : 특정 사용자만 사용
    • System install : OS 사용자 모두 사용

위와 같으므로 각자의 처지에 맞게 설치한다. Insiders version은 일종의 beta version과 같은 것으로써 내 경우 이 것을 사용해 봤는데 단축키가 먹지 않아 지웠다.

Live Sass Compiler와 Live Server 확장 프로그램 설치하기

VSCode가 설치되어 있다면 Live Sass Compiler Website로 가서 상단의 'Install' button을 눌러 설치한다.

Live Server 설치

VSCode의 extension인 Live Sass Compiler를 설치하면 'Live Server' extension을 같이 설치하라는 메시지를 보게되는데, 그 이유는 Sass Compiler가 Live Server에 종속되어 있는 관계이기 때문이다. 즉, Live Server가 설치되어 있어야 Sass Compiler를 설치할 수 있다는 말인데, 이 것은 Chrome이나 Photoshop 등의 응용프로그램을 설치하기 위해서는, 먼저 Windows 등의 OS를 설치해야 되는 것 같은 관계이기 때문이다. 대개의 extension들이 그렇지만 설치한 extension을 작동시키려면 '(Reload)다시로드' 버튼을 눌러 VSCode를 재시작 한다.

이 확장은 라이브 브라우저를 다시로드하기 위해 Live Server 확장에 종속됩니다 .

Extension을 여러 개 설치할 경우 각각의 extension을 설치할 때 마다 매번 reload하지 않아도 된다. 필요한 extension을 모두 설치한 뒤 reload하면 모든 extension이 일괄 적용되어 extension들이 활성화 된다.

기존 CSS사용자와 초보자들이 SASS를 쓸 때 얻는 혜택

새로운 지식을 배우라고 권할 때, 그냥 '무지 좋다' , '안 쓰면 후회할 걸' 등 그저 막연한 표현을 쓰는 것 보다, 실제 얻을 수 있는 혜택이 뭔지 구체적으로 확인하는 것이 감이 빨리 올 것이므로, 지금부터 그 혜택이 뭔지 한가지 예를 들어보자.

Vendor prefix 자동으로 붙이기

SASS를 사용해서 누릴 수 있는 혜택은 많다. CSS 작업을 할 때 일명 '노가다'라 불리는 귀찮고 보람도 없는 작업을 해야 될 경우가 종종 있다. 그 중 하나가 모든 브라우저에서 똑 같이 보이도록, 브라우저 제조 업체별 호환을 위해 접두사(prefix)를 붙이는 작업이다. 예를 들자면 아래와 같은 것이다.

  • -webkit- : Safari, Chrome, Android, iOs, Webkit 엔진을 이용하는 브라우저
  • -moz- : Firefox, 모질라 엔진을 사용하는 브라우저
  • -ms- : Internet Explorer (아닐 때도 있음.)
  • -o- : Opera

즉, 위와 같은 접두사를 붙여야 대부분의 브라우저에 같은 모양으로 표시 되는데, 이런 단순작업은 지루할 뿐더러, 가끔 잊어 먹고 빠뜨리는 경우 몹시 짜증스럽기도 하다.

이하 동영상

SASS Compiler 환경 setting 하기

상단 Main menu에서, File > Preferences > Setting을 선택한다.

상단 검색란에 'liveSassCompile.settings.formats'을 입력하면 아래와 같은 내용 - 이하 동영상

"liveSassCompile.settings.formats": [{
"format": "expanded",
"extensionName": ".css",
"savePath": "null"
}],

/css : 현재 작업폴더의 root 레벨에 css폴더를 만들고, 그 폴더 안에 모든 .css 파일 생성
~/../css : sass파일이 있는 폴더와 '같은 레벨'에 css 폴더를 만들고 그 폴더 안에 .css 파일 생성
~//css : sass파일이 있는 폴더의 하위 레벨에 css폴더를 만들고, 그 폴더 안에 .css 생성

첫 번째 compile 한 후에 '전처리'의 의미에 대해 다시 상기 시킨다.
