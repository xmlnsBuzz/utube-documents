문법에 맞는 HTML5+ 문서

표준을 지켜서 좋은 점.

  • HTML 문법에 맞게 문서의 구조를 만들면 일관성을 유지하므로 문서의 수정 등 관리가 용이하다.
  • 시각적 media(각종 브라우저)뿐 아니라 소리, 점자, 프린터 등의 매체에서
  • 검색엔진 Crawler에 최적화(SEO : Search Engine Optimization)되어 문서 로딩시간이 빠르고, 검색할 때 우선적으로 검색된다(네이버 등은 제외).
  • 문서가 compact하므로 불필요한 데이터를 소모하지 않으므로 호스팅 비용 절약.
  • 각종 브라우저(PC, mobile등)에서 호환되므로 일관된 외형을 가진다.

HTML 문법을 따르는 문서의 규칙

A conformant document in the HTML syntax must consist of the following parts, in the following order:

  1. BOM(Byte Order Mark) 문자. (Optional).
  2. 주석과 공백문자 (Zero or More)
  3. A doctype 선언
  4. Locale 지정 (html의 lang attribute)
  5. Encoding 선언 (meta의 charset attribute)
  6. Document's Title(title element)
  7. 주석과 공백문자 (Zero or More)
  8. html element와 해당 attributes (있을 경우) and its contents(head, body, 주석 등등 ).
  9. 주석과 공백문자 (Zero or More)
